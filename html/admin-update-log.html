콘텐츠·결제·최적화 정리

파일 정리, 최적화, 복구 작업 반영

  • 업데이트 로그 페이지를 날짜별 아코디언 구조로 바꿔 한 화면에 모든 변경 내용이 한꺼번에 펼쳐지지 않도록 정리했습니다.
  • 2차 리팩토링으로 `AiLeadersUtils` 공통 유틸을 추가하고, 스토어/렌더러에 반복되어 있던 복제, 문자열 정리, 숫자 변환, 리스트 파싱, 날짜 포맷, HTML escape, 구독 알림 로직을 공통화했습니다.
  • `courses-store.js`, `applications-store.js`, `form-submissions-store.js`, `site-content-store.js`, `site-content-renderer.js`의 중복 유틸 본문을 줄이고, 외부에서 쓰는 Store API 이름은 그대로 유지했습니다.
  • 2차 리팩토링 대상 JS 총량은 90,145자에서 89,168자로 줄였고, `node --check`, 스크립트 로드 순서, 정적 자산 참조, 대표 페이지 HTTP 200 응답을 확인했습니다.
  • 불필요한 비교/임시 폴더, 중복 파비콘, 미사용 강연 아이콘, 미사용 썸네일, 미사용 영상 파일을 정리하고 삭제 전 `backup/20260708_164500_before_cleanup_unneeded_files`에 백업했습니다.
  • 랜딩페이지에 직접 들어가 있던 base64 이미지 16개를 실제 이미지 파일로 분리해 `html/index.html` 용량을 약 9.88MB에서 약 189KB로 줄였습니다.

  • `html/assets/site-content-renderer.js`에서 더 이상 사용하지 않는 `renderHeroLegacy()` 함수를 제거했습니다.
  • `html/admin-site-content.html`의 파비콘 경로를 실제 공통 파비콘 위치인 `../favicon.ico`로 수정했습니다.
  • `shared.css`는 16개 하위 페이지가 사용하는 공통 CSS로 확인되어 복구하고 유지 대상으로 재분류했습니다.

  • 대표 페이지 CSS 경로, 공통 CSS HTTP 응답, 랜딩페이지 HTML, 관리자 콘텐츠 스크립트 문법 검사를 다시 확인했습니다.

관리자·카드·모바일 정비

관리자 구조와 강연 등록 화면 정비

  • 관리자 페이지 상단과 사이드 구조를 정리하고, 대시보드를 기본 진입 화면으로 맞췄습니다.
  • 강연 관리 화면을 목록 중심으로 바꾸고, 새 강연 생성과 수정은 별도 편집 화면으로 분리했습니다.
  • 강연 목록은 한 번에 10개씩 보이도록 하고, 이전·다음 버튼으로 넘기는 페이지 구조를 넣었습니다.
  • 새 강연 생성과 수정 화면에 강연명, 구분, 카테고리, 강사, 장소, 상세 주소, 강연일, 시작·종료 시간, 신청 시작일, 신청 마감일, 정원, 신청자 수, 가격, 카드 배지, 요약 설명 입력을 연결했습니다.
  • 카테고리는 기본 옵션 선택과 직접 입력을 같이 지원하도록 바꿨고, 강사도 아이온·문건우 선택과 직접 입력을 같이 지원하도록 정리했습니다.
  • 썸네일과 상세 이미지는 드래그 앤 드롭, 파일 첨부, 경로 직접 입력이 모두 가능하도록 만들고, 정사각 썸네일과 세로형 상세 이미지가 미리보기에서 다 보이게 조정했습니다.
  • 강연 시간은 매번 직접 입력하지 않도록 30분 단위 선택 방식으로 바꾸고, 무료 강연 기본 정원 80명, 유료 강연 기본 정원 30명 규칙을 연결했습니다.

강연 데이터 연동과 카드 표시 규칙 정리

  • 대표 강연, 무료 강연, 유료 강연 카드가 관리자 데이터 기준으로 제목, 강사명, 신청자 수, 잔여석, 카드 배지, 이미지, 강의 일정 정보를 읽도록 연결했습니다.
  • 강사명은 카드 제목 아래에 표시하고, 강사명 뒤에 `강사` 고정 문구가 붙도록 정리했습니다.
  • 잔여석은 카드 내에서 숫자만 보이도록 바꾸고, 10명 이하이면 `마감 임박` 배지가 붙게 했으며, 기간 만료 카드에서는 잔여석이 노출되지 않도록 처리했습니다.
  • 잔여석은 어떤 경우에도 3명 미만으로 내려가지 않도록 맞췄고, 상세 페이지 신청자 수도 그 규칙에 맞춰 계산되도록 정리했습니다.
  • 상세 페이지 신청자 수는 `신청자 / 정원` 형태로 바꾸고, 제목 아래 정보 카드에 강사명 항목도 추가했습니다.
  • 상세 페이지의 장소 표시는 관리자 `상세 장소/주소`, 제목의 `[지역명]` 표시는 관리자 `장소명/지역`을 따르도록 구분했습니다.
  • 상세 페이지 우측 신청 카드의 수량 선택 UI는 1인 1신청 구조와 맞지 않아 제거했습니다.

정렬, 자동 노출, 신청 데이터 규칙 정리

  • 대표 강연은 무료·유료 강연 데이터 중 신청 기간이 남은 강의를 우선 반영하고, 신청자가 많은 순서와 기간 기준 정렬이 맞도록 구조를 정리했습니다.
  • 기간이 남은 강의는 마감이 가까운 순서로, 기간이 지난 강의는 최근 날짜 순서로 뒤에 배치되도록 카드 정렬 규칙을 수정했습니다.
  • 유료 강연과 무료 강연 목록은 PC에서 6개, 모바일에서 3개씩 보이고, 페이지를 넘길 때 첫 번째 카드가 잘 보이는 위치로 자동 이동되도록 보정했습니다.
  • 강연이 신청될 때 신청자 수가 1명씩 증가하고, 관리자에서 삭제할 때는 유지 삭제와 되돌림 삭제가 구분되도록 신청 관리 흐름을 정리했습니다.
  • 관리자 신청 데이터 저장 형식을 JSON이 아니라 엑셀 내보내기 기준으로 정리하고, 무료 강연·유료 강연 신청, 출강 문의, 강사 지원 데이터를 각각 관리할 수 있게 나눴습니다.

문의·지원 폼과 첨부 파일 흐름 정리

  • 출강 문의 폼에 지역과 선호 강사 항목을 추가하고, 관리자 페이지에서 해당 값까지 확인할 수 있도록 연결했습니다.
  • 강사 지원 폼에서는 포트폴리오와 이력서 첨부를 드래그 앤 드롭으로도 올릴 수 있게 했고, 100MB 초과 시에는 링크로 받는 기준을 맞췄습니다.
  • 강사 지원 첨부 파일은 관리자에서 내려받을 수 있는 흐름까지 같이 정리했습니다.

모바일 반응형과 내비게이션 수정

  • 랜딩과 서브 페이지의 햄버거 버튼 위치를 우측 상단 기준으로 통일하고, 하위 카테고리는 기본 닫힘 상태에서 눌렀을 때 열리도록 바꿨습니다.
  • 대표 강연 섹션은 모바일에서 좌우 이동 버튼과 드래그 이동을 같이 지원하도록 만들고, 일정 시간마다 자동으로 다음 페이지로 넘어가게 했습니다.
  • 강사진 섹션 모바일 화살표는 이미지 양옆에 위치하도록 다시 잡고, 눌렀을 때가 아니라 PC처럼 버튼 자체만 색이 반응하도록 수정했습니다.
  • 무료 강연, 유료 강연, 후기, 배너, 히어로, 상세 페이지 썸네일 위치까지 포함해 여러 반응형 깨짐 문제를 순차적으로 보정했습니다.

이 날짜의 작업은 관리자 페이지 기반 구조를 잡고, 공개 페이지와 데이터가 같은 기준으로 움직이게 만드는 데 초점을 맞췄습니다.
